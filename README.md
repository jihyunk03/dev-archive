# 📚 dev-archive

> A collaborative programming study repository for continuous learning and documentation

## 🎯 Goals

**규칙적으로 공부하며, 문서화하자.**

서로가 다른 언어를 공부할 수도, 같은 언어를 공부할 수도 있으니 최대한 문서화하며 기록을 남기고 함께 공부합시다 :)

## 📋 Rules

### File Structure
- `분야명/` 폴더 하위에 마크다운 파일 업로드
- 파일 형식은 자유롭게 작성
- 파일명 마지막에 `_본인이름` 포함
  ```
  예시: java/spring-basic_김지현.md
        python/django-tutorial_신은영.md
        cs/algorithm-study_Eunice.md
        database/mysql-optimization_Jihyun.md
  ```

### Workflow
1. **개인 브랜치**에서 작업
2. **Pull Request** 생성
3. **상호 코드 리뷰** 진행
4. 리뷰 완료 후 **서로가 머지**

### Study Schedule
- 공부 일정은 각자 알아서 관리
- 최대한 **꾸준하게** 진행하기

## ✍️ Writing Guide

**처음 보는 사람도 최대한 이해할 수 있도록 작성해보자.**

하지만 보이기 위함보다는 **본인의 공부를 위함**이 가장 큰 목적이어야 한다.

### Recommended Elements
- 학습한 개념 정리
- 코드 예제와 설명
- 개인적인 이해와 해석
- 참고 자료 링크
- 궁금했던 점과 해결 과정

## 🚀 Getting Started

1. 레포지토리 클론
2. 개인 브랜치 생성
3. `분야명/` 폴더에 학습 내용 작성
4. PR 올리고 리뷰 요청
5. 함께 성장하기!
